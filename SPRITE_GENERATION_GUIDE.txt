ADVERSARY — Sprite Generation Guide
=====================================

Tool: Pixellab.ai (free, built for game pixel art)
Backup: Microsoft Copilot Image Creator (free, uses DALL-E 3)
Cleanup: Piskel (piskelapp.com, free, browser-based pixel editor)

All sprites target the EARLY NES era (1983-1985) — Donkey Kong, Ice Climber,
Wrecking Crew. Chunky proportions, big heads, stubby legs, 2-3 colors per
sprite plus transparency, no anti-aliasing, no dithering, hard pixel edges.

=======================================================================
CLEANUP STEPS (after AI generation)
=======================================================================

1. OPEN PISKEL (piskelapp.com) — free, browser-based pixel editor.

2. IMPORT the AI-generated image into Piskel:
   - File > Import > Open from computer
   - Or paste/drag the image in

3. SEPARATE FRAMES — If the AI gave you a sheet, use Piskel's "Import
   spritesheet" option:
   - Set frame width/height (16x16 or 8x8 depending on sheet)
   - Piskel will split it into individual editable frames

4. FIX STRAY PIXELS — Zoom in to 800%+ and clean up:
   - Remove any anti-aliasing (smooth gradients between colors)
   - Delete stray pixels that don't belong
   - Sharpen edges — every pixel should be deliberate
   - Make sure transparency is clean (no semi-transparent pixels)

5. LOCK TO NES PALETTE — The NES had 54 hardware colors. You don't need
   to use all of them, but every color in your sprite should be one of
   these. The key colors you'll use most:

   REDS:      #7C1F00  #BC2F00  #FC3838  #FCBCBC
   ORANGES:   #A44000  #E47818  #FCC07C
   YELLOWS:   #AC7C00  #FCC800  #FCF87C
   GREENS:    #005800  #00A800  #00FC00  #B8F8B8
   BLUES:     #0000A8  #0058F8  #3CBCFC  #A4E4FC
   PURPLES:   #44009C  #6844FC  #B8B8F8
   GRAYS:     #080808  #3C3C3C  #7C7C7C  #BCBCBC  #FCFCFC
   SKIN:      #E4A484  #FCC4A8
   BROWNS:    #503000  #8C4C18  #C89050

   In Piskel, you can set up a custom palette with just these colors
   so you don't accidentally use off-palette shades.

6. CHECK FRAME SIZE — Make sure each frame is exactly the right pixel
   dimensions (16x16, 16x24 for boss, 8x8 for items/UI). Resize canvas
   in Piskel if the AI got it slightly wrong.

7. TEST ANIMATION — Piskel has a live preview panel. Set the FPS to
   about 8-10 (NES typically ran animations at 6-12 fps) and check:
   - Does the walk cycle waddle look good?
   - Are attack windups readable?
   - Do death animations look clear?

8. EXPORT — When a sheet is cleaned up:
   - Export as PNG spritesheet (File > Export > PNG spritesheet)
   - Set columns to match your frame count
   - Save to the project's assets/ folder
   - Name files clearly: player.png, hollow_soldier.png, tileset_L1.png, etc.

9. REPEAT for each of the 13 prompt sheets.
